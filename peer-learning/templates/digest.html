01 Security · cross-service auth

A Key of Its Own

Ian Gardiner · USPDS-404 · web + zenpayroll

When DSA chat moved onto Salesforce's embedded messaging (MIAW), Gusto had to prove to Salesforce that the person in the chat is who we say they are. The move worth stealing isn't "mint a JWT" — it's minting it with a dedicated signing key that lives completely apart from the org's JWKS.

The token is a short-lived RS256 JWT — sub is the UserRole UUID, signed with a MIAW-only key and served from its own public endpoint (/directed_support/miaw_jwks): unauthenticated, public keys only. The telling move: Ian deliberately duplicated ~15 lines of JwkHelper instead of parameterizing the shared signing infra — that helper powers auth, gRPC, and OAuth org-wide, so widening it for one feature is the wrong blast radius. Isolation is the feature, not a shortcut.

↳ This token lands as dsaChatIdentityToken on the same SupportInfo GraphQL type your USPDS-677 branch is extending right now — same file, same object.
Why it matters — when a new audience needs to verify identity, give it its own key and its own endpoint; don't widen the security infrastructure half the org already leans on.

02 Data architecture

The Fingerprint, Not the Photograph

Kilian O'Donnell · USPDS-622 · zenpayroll

The DSA dashboard hides a company once an advisor has acted on it — but only while the underlying data still matches what they saw. The old way stored the entire company snapshot and, on every read, deserialized it and re-hashed it to compare. This sprint that whole dance became a single indexed string compare.

The arc is a clean expand → migrate → contract, one PR per beat: add a nullable company_data_hash column; start stamping and persisting the hash on write while reads still go the old way; flip the hide filter to compare the stored hash directly — no JSON deserialize, no re-canonicalize; then drop the now-redundant hashes from the response payload. Store the fingerprint, not the photograph.

Why it matters — when all you need to know is "did this change?", persist a hash and compare strings — don't re-derive the comparison from a fat snapshot on every read.
Edge case worth keeping: canonicalization had to coerce whole-number Floats to Integer — JS's JSON.stringify drops the trailing .0, so 500000.0 on the server and 500000 after a browser round-trip would otherwise never match, and every prior hide would silently stop matching.
Expand → migrate → contract, one PR per step

03 Security · input handling

Basename Before You Build a Path

Prudhvi Avula · PSVULN-827 · zenpayroll

An uploaded attachment's filename is attacker-controlled. Join it straight into a temp path and a name like ../escaped.txt writes outside the directory you meant. One line closes it.

Run the original filename through File.basename before File.join — it strips every directory component, so the write lands inside tmp_dir no matter what the client called the file. Names that reduce to nothing useful (., .., empty) fall back to a safe default. The spec earns its keep: it actually attempts the escape and asserts the file lands inside the sandbox, not above it.

Why it matters — any time an untrusted string becomes part of a filesystem path, File.basename it first. The fix is one line; the test that proves it is three.
