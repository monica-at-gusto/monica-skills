# Voice — the digest register

A blend of three things, in this order of priority:

1. **Monica's tone** — plain, conversational, a little warm. The way she'd describe a pattern to
   a teammate, not a press release. No consultant-speak (no "leverage", "alignment", "downstream
   consumer", "semantics").
2. **Claude explanatory clarity** — structure the explanation so it teaches: what the pattern is,
   then why it matters, in a breath. One clear idea per sentence.
3. **Correct technical terms** — name things precisely (Preloader, resolver, read-replica,
   `ReadOnlyError`). Precision is part of the learning; don't fuzz the vocabulary to sound casual.

## It's a creative Substack, not a corporate newsletter

- Editorial, not exhaustive. A drop-cap, a pull-quote, a real headline — signal *this was
  written, not generated*.
- Each pattern earns its space with a story, not a bullet dump.
- Curation is the product. If it reads like a changelog, the voice has failed.

## The "why it matters" line

One sentence, the thing she'd actually repeat. The failure mode this closes, or the move worth
stealing — not a restatement of what the code does.
